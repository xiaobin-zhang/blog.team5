java8特性介绍

发表于 2018-03-21 更新于 2023-02-22 分类于 编程

java8概述

java8 初探，本篇文章主要介绍java8部分新特性的定义和简单使用，lambda表达式、函数式接口、接口的默认方法和静态方法、Stream流、时间处理函数。其中，lambda语法是java的一个重大突破，在学习java8的时候，大家要首先要转变的是自己一贯的面相对象的思维。在本章节的介绍中，也包含了一点目前流行的函数式编程的一些概念。java8的出现可以使java支持函数式编程。听起来是不是有点高大上呢？除了lambda语法，还有好多有趣的新特性，比如Stream流等。那么，接下来，就让我们一起来看一下java8的新特性到底是个什么样。

主要内容

lambda表达式

定义

Lambda 表达式：

(lambda expression)是一个匿名函数，Lambda表达式基于数学中的λ演算得名，直接对应于其中的lambda抽象(lambda abstraction)，是一个匿名函数，即没有函数名的函数。Lambda表达式可以表示闭包（注意和数学传统意义上的不同）。

官方解释：

Lambda expressions. Funda-men-tally, a lambda expression is just a shorter way of writing an implementation of a method for later execution

lambda表达式和函数式接口和结合使用，使java可以支持函数式编程。我们首先不要把lambda当做是类似于Date、List等等java的普通api那样使用，不要把它理解为像调用普通方法一样使用，lambda表达式需要和函数式接口一起使用。

函数式接口：FunctionalInterface

定义

  • 注解：@FunctionalInterface
  • 函数式接口不允许是空接口,也不允许只有默认方法和静态方法
  • 如果有抽象方法，则只能有一个抽象方法
  • 抽象方法和普通方法在函数式接口中不允许共存
  • 可以有多个默认方法，默认方法必须要有方法体
  • 可以有多个静态方法，静态方法必须要有方法体,静态方法只能通过接口调用

使用

  • 实现类必须重写接口的抽象方法

  • 如果实现类实现了多个函数式接口，也必须重写全部接口的抽象方法

接口的默认方法和静态方法

默认方法

定义

  • 接口中的默认方法，必须用default关键字修饰

  • 接口中的默认方法，必须有方法体

使用

  • 如果多个函数式接口中存在相同的默认方法，则必须重写该默认方法，

  • 在重写默认方法时，可以指定使用哪个接口中的方法，也可以全部不用，完全重写

静态方法

定义

  • 接口中的静态方法，必须用static关键字修饰

  • 接口中的静态方法，必须有方法体

使用

  • 静态方法可以直接使用接口调用

Stream

定义

  • Sequence of elements: A stream provides an interface to a sequenced set of values of a specific element type. However, streams don’t actually store elements; they are computed on demand
  • Source: Streams consume from a data-providing source such as collections, arrays, or I/O resources.
  • Aggregate operations: Streams support SQL-like operations and common operations from functional programing languages, such as filter, map, reduce, find, match, sorted, and so on.
    此外，流操作有两个基本特征，使它们与收集操作非常不同：
  • Pipelining: Many stream operations return a stream themselves. This allows operations to be chained to form a larger pipeline. This enables certain optimizations, such as laziness and short-circuiting, which we explore later.
  • Internal iteration: In contrast to collections, which are iterated explicitly (external iteration), stream operations do the iteration behind the scenes for you.
@Raoul-Gabriel Urmawww.oracle.com/technetwork/articles/java/ma14-java-se-8-streams-2177646.html

翻译：

  • 元素的序列: 元素是特定类型的对象，形成一个队列。 Java中的Stream并不会存储元素，而是按需计算。
  • 数据源: 流的来源。 可以是集合，数组，I/O channel， 产生器generator 等。
  • 聚合操作: 类似SQL语句一样的操作， 比如filter, map, reduce, find, match, sorted等。
    和以前的Collection操作不同， Stream操作还有两个基础的特征：
  • Pipelining: 中间操作都会返回流对象本身。 这样多个操作可以串联成一个管道， 如同流式风格（fluent style）。 这样做可以对操作进行优化， 比如延迟执行(laziness)和短路( short-circuiting)。
  • 内部迭代: 以前对集合遍历都是通过Iterator或者For-Each的方式, 显式的在集合外部进行迭代， 这叫做外部迭代。 Stream提供了内部迭代的方式， 通过访问者模式(Visitor)实现。
@菜鸟教程www.runoob.com/java/java8-streams.html

总的来说，Stream是用来操作集合的利器。提供了对集合操作的新的方式，并且比java8之前处理集合的方法在性能和语义上，都由很大提升。

使用

Stream在使用使主要有生成、过滤、排序、限制、去重、匹配、合并、收集、统计等。Stream对集合的操作和编写SQL语言极其相似，只要会使用SQL检索数据库，那么对于Stream的掌握会事半功倍。

流的生成

流的生成有两种方式：

  • Stream自己创建

  • 从集合生成

过滤

关键字：filter。Stream流操作集合时，可以对stream中的元素进行特定条件的过滤。可相当于SQL中Where后面的判定条件。

过滤，允许指定一个断言Predicate来从数据流中排除一些元素。

排序

关键字：sorted。对stream中元素的排序。可以自定义排序规则。相当于SQL中的order by。

排序，允许指定一个比较器Comparator来对数据流进行排序。

限制

关键字：limit。对stream中元素的限制。

关键字：skip。跳过，使用skip和limit实现分页

去重

关键字：distinct。

匹配

关键字：allMatch、anyMatch、findFirst和findAny。来检查数据集中元素的某一属性是否符合要求

合并

关键字：flatMap、map。在stream过程中，对产生的分支stream合并为一个stream。
需要一个实现了Function<? super T, ? extends R>接口的lambda表达式来将一个数据流转化为另一个数据流.

收集

关键字：collect。对stream中元素的收集，这是一个结束操作.collect(Collectors.toList())将结果集中的信息放入列表

统计

关键字：count。

时间处理函数

在java8之前，我们在处理日期时，一般使用的是Date或Calendar。在对日期的格式化时，我们用SimpleDateFormat。除此之外，也使用过org.joda.time.LocalDate来处理日期。
在使用过程中，发现，使用org.joda.time.LocalDate处理日期比较方便。在java8的新特性中，同样采用了org.joda.time的设计概念，推出了java.time.LocalDate来进一步加强对日期与时间的处理。

java8之前日期处理弊端

  • 在旧版的 Java 中，日期时间 API 存在诸多问题，其中有：
  • 非线程安全 − java.util.Date 是非线程安全的，所有的日期类都是可变的，这是Java日期类最大的问题之一。
  • 设计很差 − Java的日期/时间类的定义并不一致，在java.util和java.sql的包中都有日期类，此外用于格式化和解析的类在java.text包中定义。java.util.Date同时包含日期和时间，而java.sql.Date仅包含日期，将其纳入java.sql包并不合理。另外这两个类都有相同的名字，这本身就是一个非常糟糕的设计。
  • 时区处理麻烦 − 日期类并不提供国际化，没有时区支持，因此Java引入了java.util.Calendar和java.util.TimeZone类，但他们同样存在上述所有的问题。

Date Time API

  • LocalDate - 日期函数
  • LocalTime - 时间函数
  • LocalDateTime - 日期+时间函数
  • DateTimeFormatter - 日期时间格式化函数
